# Agri-Ergianto-Prasetya---Portfolio

# 🧠 Brainstorming Session

**Tanggal:** 5 Mei 2026
**Tujuan:** Menentukan konsep website portofolio yang profesional namun tetap personal.

## Ide Utama
- **Tema:** Minimalis dan Elegan.
- **Warna Utama:** Hitam, Putih, dan sedikit aksen Biru/Emas untuk kesan profesional.
- **Elemen Unik:** Menggunakan metafora "Kucing dan Singa" untuk menjelaskan perkembangan karakter (proses belajar dari pemalu menjadi pemimpin/trainer).

## Fitur yang Direncanakan
1. **Profil:** Perkenalan singkat sebagai Computer Engineering graduate.
2. **Proyek:** Penjelasan mengenai proyek yang sudah dibuat.
3. **Experience/Skills:** Daftar tech stack (Figma, Canva, Java, Cloud Computing).
4. **Contact:** Link LinkedIn, Gmail, dan GitHub.

## Hasil Konsultasi AI
- AI menyarankan penggunaan semantic HTML5 untuk aksesibilitas yang lebih baik.
- Saran untuk menggunakan pendekatan "Mobile-First" agar website rapi di smartphone.

# 📝 Project Details (RTCC-O Framework)

| Component | Detail |
| :--- | :--- |
| **Role** | Senior Front-End Developer & UI/UX Expert. |
| **Task** | Membuat struktur HTML5 semantik dan styling CSS modern untuk portofolio satu halaman. |
| **Context** | Project ini adalah portofolio untuk Agri Ergianto Prasetya, lulusan Teknik Komputer yang memiliki minat di Trainer & Tech. |
| **Constraints** | Wajib menggunakan HTML5 murni & CSS (no framework seperti Bootstrap dulu), harus responsif, dan kode harus bersih. |
| **Output** | File `index.html` dan `style.css` yang siap dideploy ke GitHub Pages. |

## Rencana Struktur HTML
- `<header>`: Navigasi.
- `<main>`: Profil, Proyek, Skills.
- `<footer>`: Copyright & Social Media.

# 🛠️ Step-by-Step Execution

1. **Inisialisasi Project:** Membuat repository di GitHub dan melakukan clone ke lokal.
2. **Setup Folder:** Menyiapkan folder `plan/` dan `assets/`.
3. **Coding HTML:** Menyusun struktur dasar berdasarkan rencana di `02-details.md`.
4. **Styling CSS:** 
   - Menerapkan reset CSS.
   - Mengatur tipografi (font Inter/Poppins).
   - Membuat layout responsif dengan Flexbox/Grid.
5. **Testing:** Mengecek tampilan di browser Chrome (Desktop & Mobile).
6. **Deployment:** Melakukan `git push` dan mengaktifkan GitHub Pages.
